    git使用命令

    xiaofan 5/22/2022

    # 项目中常用的git命令

    # 打开终端，切换到项目根目录中，初始化本地git仓库

    如果你是自己想推送到自己的仓库或其他的仓库 可以在没有git依赖下 运行以下 - 拉取下来的项目一般很少要

    git init
    1

    # 推荐ssh 拉取项目

    下面的为ssh链接拉取，在仓库主页的克隆/下载点击弹出层选择ssh 前提是你要配置了密钥

    如果你没有根据自己的git仓库来进行第一次推送，那么你拉取的默认分支是你第一次推送的分支名

    • 终端或者git环境运行
    git clone git@gitee.com:xxx-xxx-xxx/xxx.git
    1

    # 拉取单独分支

    命令格式如下，-b后面接分支名，最后是git地址。 此方式是ssh拉取

    git clone -b XXX git@gitee.com:xxx-xxx-ixxxk/xxx.git
    1

    # 查看当前项目中所有的分支

    默认是你当前在编辑的分支

    git branch
    1

    # 检查当前项目的文件状态

    红色是未提交。绿色是已提交

    git status
    1

    # 在主分支下创建一个子分支

    新创建成功的子分支会默认为当前的分支 。 即执行 git branch 会出现 👇 (分支顺序不重要)

    git checkout  -b 分支名 

--- * 当前分支名
--- master - 主分支
--- *** - 其他分支
    1
    2
    3
    4
    5

    • # 本地提交 - 将本地的home分支进行本地的comm提交

    # 本地的commit提交

    git add .  -- 将所有新增的、修改过后 的文件都加入到本地暂存区
git commit -m "完成了**功能的开发"  -- 本地提交。双引号要包着消息
    1
    2

    # 将本地的 ** 分支推送到远程仓库进行保存

    git push -u origin 分支名   -- 将本地 ** 分支推送到远程仓库
    1

    # 将本地的 ** 分支合并到本地的master 分支

    建议运行之后再运行命令 git branch 检查当前分支的状态

    git checkout master  --切换到master分支上。
    1

    # 合并 分支代码

    这里的分支根据👆方切换到的分支，注意，一般都是在子分支把全部功能开发完毕后测试完成后再推送到主分支

    git merge 分支名
    1

    # 将最新的代码推送到码云仓库

    git push
    1

    # 删除 已经推送完成的 ** 分支

    当前代码一定一定要确定当前分支不为master分支!!!!!!

    git branch -d 分支名
