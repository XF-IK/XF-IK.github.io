    # 对于Vue是一套渐进式框架的理解?

    渐进式代表的含义：你想用或者能用的功能特性，你不想用的部分可以不用，Vue不强求你一次性接收并使用他的全部功能特性

    • 场景一：公司刚开始一个项目，技术人员对Vue的掌握也不足够。那么我们就不能使用Vue了吗？ 当然不是，如果你只是使用 Vue的那些基础操作，如：页面渲染、表单处理提交功能，那还是非常简单的，成熟技术人员上手也就一两天，。完全可以使用它代替jquery。 并不需要你去引入其他复杂性功能。
    • 我们的项目规模逐渐变大了，我们可能会逐渐用到前端路由、状态集中管理、并最终实现一个高度工程化的前端项目。这些功能特性我们可以逐步进行引入，当然不用也可以。 -- 所以Vue的适用面很广，你可以用它代替老项目中的jquery，也可以在新项目启动初期，有限的使用功能特性，从而降低上手的成本。

    没有多做职责之外的事情

    • vue.js 只提供了 vue-cli 生态中最核心的 组件系统 和 双向数据绑定
    • 像 vuex、vue-router都属于围绕vue.js开发的库。

    比如说，你要使用Angular，必须接收以下东西：

    • 必须使用他的模块机制
    • 必须使用他的依赖注入
    • 必须使用他的特殊形式定义组件（这一点每个视图框架都有，难以避免） 所以Angular 是带有比较强的排它性的，如果你的应用不是从头开始，而是要不断考虑是否跟其他东西集成，这些主张会带来一些困扰。

    比如说，你要使用React，你必须要理解：

    • 函数式编程的理念
    • 需要知道什么是副作用
    • 什么是纯函数
    • 如何隔离副作用
    • 他的侵入性看似没有Angular那么强，主要因为它是软性侵入

    Vue与React、Angular的不同是，它是渐进的：

    • 你可以在原有的大系统上面，把一两个组件改用它实现，当jQuery用
    • 也可以整个用它全家桶开发，当Angular用
    • 还可以用他的视图，搭配闭包自己设计的整个下层用
    • 你可以在底层数据逻辑的地方用OO和设计模式的那套理念，
    • 也可以函数式，都可以，它只是个轻量视图而已，只做了最核心的东西。

    # vue.js的两个核心是什么?

    数据驱动，也叫双向数据绑定。

    • 在vue中，数据的改变会驱动视图的自动更新，核心是VM，即是ViewModel，是指视图是由数据驱动构成的，我们对视图的修改，不会直接操作DOM，而是通过修改数据，保证了数据和视图的一致性。传统的做法是需要手动改变DOM来使得视图更新，而vue只需要改变数据
    • vue数据观测原理在技术实现上，利用的是ES5Object.defineProperty和存储器属性：getter和setter

    组件系统

    • 组件的出现就是为了解决页面布局等等一系列问题，而vue中的组件分为两种，全局组件和局部组件，它提供了强大的页面布局功能。
    • 组件化开发，优点很多，可以很好的降低数据之间的耦合度，将常用的代码封装成组件之后 [vue组件的封装方法]，就能高度的复用，提高代码的可用性。一个页面/模块可以由多个组件所组成。
    - vue组件的核心选项：👇
- 模板（template）：模板声明了数据和最终展现给用户的DOM之间的映射关系。

- 初始数据（data）：一个组件的初始数据状态。对于可复用性的组件来说，这通常是私有状态。

- 接收的外部参数（props）：组件之间通过参数来进行数据的传递和共享。

- 方法（methods）：对数据的改动操作一般都在组建的方法内进行。

- 生命周期钩子函数（lifecycle hooks）：一个组件会触发多个生命周期钩子函数，最新2.0版本对于生命周期函数名称改动很大

- 私有资源（assets）：Vue当中将用户自定义的指令、过滤器、组件等统称为资源。一个组件可以声明自己的私有资源、
私有资源只有该组件和它的子组件可以调用

    # v-show 和 v-if 的区别?

    • v-show 是通过控制 display:hidden 来控制显示和隐藏
    • v-if 是通过组件的创建和销毁实现显示和隐藏

    # Vue的常用修饰符有哪些?

    # 事件修饰符

    • .stop 阻止事件继续传播(冒泡)
    • .prevent 阻止标签默认行为，相当于调用了event.preventDefault方法
    • .capture 使用事件捕获模式，即元素自身触发的事件先在此处理，然后才交给内部元素进行处理
    • .self 只当在 event.target 是当前元素自身时触发处理函数
    • .once 事件将只触发一次
    • .passive 告诉浏览器你不想阻止事件的默认行为,在移动端上能提升移动端的性能

    使用修饰符时，顺序很重要；相应的代码会以同样的顺序产生。因此，用 v-on:click.prevent.self 会阻止所有的点击
    而 v-on:click.self.prevent 只会阻止对元素自身的点击

    # v-model的修饰符

    • .lazy 默认情况下，v-model同步输入框的值和数据，可以通过这个修饰符，转变为在change事件再同步
    • .number 自动将用户的输入值转为数值类型
    • .trim 自动过滤用户输入的首位空格

    # element的修饰符 (面试回答加分🤔)
